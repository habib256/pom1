## 🆕 Candidate new programs

- [ ] **Applesoft Lite arcade pack** `[S · solid]` — leverage `GET A$` (no-Enter input) for 1977-style scroll-text arcade games: *Hammurabi*, *Lunar Lander*, *Star Trek* shoot-em-up. Pure scroll-text (no `HOME`/`VTAB`) — design for it. Save as `sdcard/<NAME>#F80801`. **No source projects exist yet** (the `sdcard/PLAB/ASOFT/` payloads are binary-only). First step: pick one game, create `dev/projects/asoft_<name>/` with a `.bas` source + tagged-binary build script.
- [ ] **TMS9918 Mode 2 (bitmap) demo** `[M · solid]` — `dev/lib/tms9918/tms9918m2.asm` driver ships and exports `init_vdp_g2`, `clear_bitmap`, `vdp_set_write/read`, `calc_pix_addr`, `plot_set`, `line_xy`. Only `tms9918_logo` consumes it today (text-mode bitmap rendering); 9 of 11 TMS projects are still Graphics I. Build a Mandelbrot / image viewer / vector demo to validate the driver in graphics use and seed the next wave of TMS games.
- [ ] **HGR sub-byte sprite showcases** `[S · solid]` — `dev/lib/hgr/subbyte4.inc` + `subbyte_fill.asm` shipped with the library reorg, but only `hgr_maze` exercises sub-byte masks. Pin the lib with one or two dedicated example projects (arbitrary-width sprite blitter, scrolling text on HGR).
- [ ] **microSD shell utilities** `[S · nice]` — small Wozmon-callable tools in 6502 asm: `CAT` (dump file via `$D012`), `XCOPY` (range copy with progress), `HEXDUMP`. The shipped `sdcard/PLAB/UTILS/` carries `MEMORYTEST`, `MONITOR`, `DISASSEMBLER`, `RAMCHALLENGE`, `A1ASSEMBLER` as tagged binaries — no source. Open question: extract sources or write fresh? Either way, ship under `dev/projects/sd_utils_*/` with `Makefile` + `dev/cc65/` config.
- [ ] **More CodeTank ROM bundles** `[S · nice]` — `roms/codetank/Codetank_GAME1.rom` ships a 6-game compilation (Galaga/Sokoban/Snake/Life lower jumper, Tetris/LOGO upper jumper). Build themed companions: `Codetank_DEMOS.rom` (graphics demos), `Codetank_BASIC.rom` (BASIC + tools), `Codetank_PUZZLES.rom` (Connect4/Maze3D/Sokoban variants). Drop under `roms/codetank/` for the in-app library to discover.
- [ ] **A1-IO & RTC sensor logger** `[S · nice]` — extend `a1io_rtc_clock` (display-only, reads 6 RTC regs) into a logger that samples the DS3231 temp + 8 ADC channels + 4 digital inputs every N seconds and prints a CSV row. The broadcast-protocol read path is currently inline in `RtcClock.asm`; extract it to `dev/lib/a1io/a1io_read.asm` first so the logger can share the helper.
- [ ] **GT-6144 Game of Life cycle-count writeup** `[S · nice]` — `gt6144_life` already runs; what's missing is the comparison vs `hgr10_life`. The 2102 SRAM is write-only via `$D00A`, so each generation needs a host-side shadow — surface that trade-off in both READMEs.
