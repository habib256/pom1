## 🧹 Project hygiene

- [ ] **Resolve `TODO` placeholders in `dev/projects/*/README.md`** `[S · solid]` — most project READMEs ship with literal `TODO` markers in three slots: *Recommended POM1 preset*, *POM1 → Presets → … (TODO)*, *License: TODO*. Pin each project to its canonical preset (text → preset 0/1, GEN2 HGR → preset 7, TMS9918 → preset 8 or 14, GT-6144 → its dedicated preset, A1-IO & RTC → its preset) and pick a license. Affected: `games_connect4`, `games_little_tower`, `games_maze`, `games_sokoban`, `a1io_rtc_clock`, `gt6144_hello`, `gt6144_life`, `hgr2_sierpinski`, `hgr4_mandelbrot`, `hgr10_life`, `tms9918_connect4`, `tms9918_galaga`, `tms9918_maze3d`, plus any new projects landing without a preset hint.
- [ ] **Per-project `LICENSE` headers** `[S · nice]` — once licenses are picked, add a 3-line comment header at the top of each `.asm` so a copy-pasted file carries its license. Match what's chosen in §1.
- [ ] **`dev/lib/` consistency pass** `[S · nice]` — `dev/lib/{apple1,m6502,tms9918,hgr,sokoban}/` mostly grew organically out of project sources. Audit for drift: ZP allocations colliding across libs (`apple1.inc` reserves `$00-$22`, others might re-claim), duplicated equates, `.include` paths that assume sibling dirs vs `-I ../../lib/<lib>`. Document each lib's ZP usage + clobber list at the top of its main `.inc`.

## 🆕 Candidate new programs

Concrete enough to scope, vague enough to leave to the next contributor's taste.

- [ ] **Applesoft Lite arcade pack** `[S · solid]` — leverage `GET A$` (no-Enter input) for 1977-style scroll-text arcade games: *Hammurabi*, *Lunar Lander*, *Star Trek* shoot-em-up. Pure scroll-text (no `HOME`/`VTAB` available) — design for it. Save as `sdcard/<NAME>#F80801`.
- [ ] **TMS9918 Mode 2 (bitmap) demo** `[M · solid]` — `dev/lib/tms9918/tms9918m2.asm` already implements the Mode 2 driver but no shipping project uses it. Build a small showcase (Mandelbrot in 256×192 colour, image viewer reading from microSD, vector-art demo) to validate the driver and seed the next wave of TMS games.
- [ ] **GT-6144 Game of Life** `[S · nice]` — `gt6144_life` exists but the framebuffer is on 6× Intel 2102 SRAM (write-only via `$D00A`), so each generation needs a host-side shadow. Compare cycle counts vs. HGR Life (`hgr10_life`) to surface the trade-off in the docs.
- [ ] **HGR sub-byte sprite library** `[M · solid]` — extract the maze sub-byte mask LUT pattern (currently inline in `hgr_maze`) into `dev/lib/hgr/subbyte.inc` so any project can draw arbitrary-width sprites without re-deriving the 7-phase mask tables. Pin with one or two example projects.
- [ ] **microSD shell utilities** `[S · nice]` — small Wozmon-callable tools written in 6502 asm: `CAT` (dump a file via `$D012`), `XCOPY` (range copy with progress), `HEXDUMP`. Ship under `sdcard/UTILS/` with tagged filenames.
- [ ] **CodeTank ROM bundles** `[S · nice]` — package complete TMS9918 games as standalone CodeTank 32 KB ROMs (Galaga, Tetris, Sokoban) so a real P-LAB stack with TMS9918 + CodeTank runs the program with no cassette/microSD dependency. Drop the `.rom` images under `roms/codetank/` for the in-app CodeTank Library window to discover.
- [ ] **A1-IO & RTC sensor logger** `[S · nice]` — extend `a1io_rtc_clock` from a clock display to a logger that samples the DS3231 temp + 8 ADC channels + 4 digital inputs every N seconds and prints a CSV row. Useful as a real-world demo of the broadcast protocol.
