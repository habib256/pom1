# lib/hgr — Uncle Bernie's GEN2 HGR data tables

Lookup tables for the GEN2 HGR card (passive RAM-mapped framebuffer at
`$2000-$3FFF`). The card is 280×192 NTSC artifact-color; pixel layout follows
the Apple II non-linear scanline addressing.

## Files

- **`hgr_tables.inc`** — row-address LUT mapping scanline `Y (0..191)` → base
  address inside `$2000-$3FFF`. Apple-II compatible non-linear ordering.
- **`smiley.inc`** — sample 16×16 smiley sprite, generated by
  `dev/projects/hgr9_smiley16/_gen_smiley16.py`.
- **`bbfont_cp437.inc`** — Beautiful Boot 8×8 font, full CP437 (256 glyphs,
  2 KB). Encoding: `AND #$7F`, 8 rows top→bottom, bit 0 = left. Source:
  Michael Pohoreski's `apple2_hgr_font_tutorial`. Label: `HGR8_BBFont`,
  constants `HGR8_BBFONT_BYTES_PER_GLYPH` / `HGR8_BBFONT_GLYPH_COUNT`.
  Used by `dev/projects/hgr8_bbfont_show/`.
- **`bbfont_subset.inc`** — 38-glyph subset of the same font, hand-curated
  for the HGR Sokoban HUD/title (digits, "MOVES:", "PUSHES:", "SOKOBAN",
  letters). Label: `HGR6_Sokoban_bbfont`. Used by
  `dev/projects/hgr6_sokoban/`. Drop-in compatible with `bbfont_cp437.inc`
  glyph layout.

These are data-only includes. Drop them into your `.asm` with `.include` and
reference the labels they define. See `dev/projects/hgr*/` for callers.

## Use

    .include "apple1.inc"
    .include "hgr_tables.inc"

In your project Makefile:

    LIB := -I ../../lib/apple1 -I ../../lib/hgr
    LOAD_CFG := ../../cc65/apple1_gen2.cfg
